We Should Love-Hate APL

APL is 50+ years old programming language that excels at multidimensional array manipulation.It's standard form uses many nonstandard characters. A language called J exists that synthesizes the symbols into a standard character language. APL is a mad but beautiful reminder that informatics is not necessarily about computers or science. It can also be Reason Alone and is not bond by ancient Roman letters. It forces us to think and innovate with the design of its specialized editors. Perhaps it has been incubating for the era of tablet computers where its special symbols are just a capacitive touch away instead of contrained to keyboard stickers. It also acts as a convenient notation which while mathematical in appearance retains the beauty we love in code: no ambiguity.

Lisp and Haskell

The general activities of software developers can be split into two as systems programming and application programming. These further can be split into two as construction (setting and getting) and algorithm research. Thus there are four quadrants. Most of the computing needs of humanity are systems programming and construction. This includes communication, text, images, videos, and database management. Essentially software only needs to replace paper, ink, radio, and television. Mutation is not just a result of limited memory in those domains, it is the desire. Lisp, Haskell, and the rest of the research languages are for exotic algorithms in the neatly garbage collected application layer. There has been no need for them outside of esoteric math applications(which are very honorable) until recently. To their communities I say be patient your day is coming you have labored without payment, growing these research languages in the dark like a garden of mushrooms, but good old construction is here to stay.

File Formats

In my car I listen to the recordings of a bunch of guys discussing the theory of competitive Pokemon battling. Constantly. These recordings after they happen live are uploaded as a dreaded Flash Video on YouTube. Another webpage can take the URL of this YouTube video and convert its sound into an mp3 file. This video was viewed and converted on a proprietary closed browser running on a proprietary closed operating system. Then these mp3 files were moved onto a flashdrive with a proprietary closed file manager. Then I took that flashdrive and and plugged it into the USB port of my car's audio system. It starts up insantly and has no user acccount control. This audio software in my car is proprietary and closed. So much proprietary software but what's that? It doesn't matter because the open standard file format (mp3) is supported from a standard interface (USB). USB given to us from the electric engineers. The mp3 format is the software developer's child. The lesson here is clear: Open File Formats (and storage formats for that matter) matter more than open source software. Relax and take joy in that.

Rust and Go

I've mentioned before the overly bloated software that developers have created. This is understandable given the youth of Informatics. It is a younger field that Psychology. Much like Psychology it has difficult in creating unified vocabulary and has blind schools of thought. Class based Object Oriented Programming, especially on the JVM and .NET may be one of those blind alleys. Given more time to contemplate we see the birth of Rust and Go, languages carefully crafted for a purpose. Rust for kernels and systems programming and Go for application layer software. Go does away with class OO and instead has functions associated with structures accessed with a dot (looks the same but is not). Go applications run on a bunch of little VMs instead of one big one like Java. Rust and Go imparticular have taken great influence from research languages like Haskell and Lisp. Common Lisp still has a role of course. The needs that bore them are parallelism, speed, the usual safety concerns, and internationalization(Unicode characters allowed from the get-go). Rust and Go see rapid development every day on GitHub. Is it another blind alley or the new carefully selected path?

What we can learn from the Atari Portfolio

The Atari Portfolio was the small computer that the kid in Terminator 2 used to hack an ATM. It had a monochromatic display that could only draw straight lines and characters. It booted immediately to a DOS clone and could only really make text files and Lotus inspired spreadsheets. This is an example of true computing with a focus and purpose. I guarantee that thing starts up faster than today's ultrabooks and can produce useful data with its spreadsheet program. It came out in 1989 and can still talk up to a Windows Vista computer. It didn't need color or images or even user account control. Single user. Single purpose, nothing unneeded for that purpose and it did this all fast. Additionally being a DOS clone OS it shows that not everyone wants Unix. A FOSS 3D printed clone of the Atari Portfolio is a project someone should look at.
